Export IA & Import IA

Le workflow Export / Import IA est le mode le plus universel : il prépare un dossier structuré que n'importe quelle IA Cowork peut ouvrir, travailler dessus et produire des documents — que vous réimportez ensuite dans Ordicab en un clic.

La fonctionnalité clé de ce mode est la pseudonymisation du dossier entier. Avant l'export, Ordicab remplace toutes les données personnelles du dossier (noms, adresses, emails, téléphones…) par des pseudonymes d'apparence réelle. L'IA travaille sur ces pseudonymes sans jamais voir les vraies identités. À l'import, Ordicab restitue automatiquement les données réelles dans les documents produits. La clé de correspondance ne quitte jamais votre machine.

Principe

Ordicab ne se connecte pas directement à un service IA. À la place, il prépare un dossier IA exporté — un répertoire autonome et structuré contenant :

  • les métadonnées du dossier (contacts, dates clés, références) ;
  • les textes extraits de vos documents (PDF, Word, emails…) ;
  • vos modèles de documents avec leurs routines ;
  • un fichier CLAUDE.md (ou équivalent) que l'agent IA lit au démarrage pour comprendre le contexte.

Vous ouvrez ce dossier avec votre IA Cowork de choix (Claude Code, Codex, ou tout agent compatible), lui demandez de créer des documents, puis vous réimportez les résultats dans Ordicab.

Compatibilité universelle.
Le dossier exporté est un répertoire de fichiers texte et JSON standard. Il fonctionne avec n'importe quel agent IA capable de lire et écrire des fichiers — aujourd'hui et dans le futur.

Pseudonymisation réversible — la grande innovation

Avant d'exporter, vous pouvez activer la pseudonymisation. Ordicab remplace alors toutes les données identifiantes par des alias cohérents et fictifs dans le dossier exporté.

Ce qui est pseudonymisé

  • Noms et prénoms des contacts
  • Adresses postales
  • Emails et numéros de téléphone
  • Numéros de sécurité sociale, dates de naissance
  • Toute donnée personnelle structurée dans Ordicab

Pourquoi c'est différent de l'anonymisation

AnonymisationPseudonymisation (Ordicab)
Données masquées Oui Oui — remplacées par des alias
Cohérence des alias Non garantie Oui — le même contact garde le même alias partout
L'IA peut travailler normalement Partiellement Oui — le contexte reste complet et cohérent
Restitution des vraies données Impossible Automatique à l'import
Document final Avec alias — à corriger manuellement Avec vraies données — prêt à l'emploi
La clé de la pseudonymisation reste sur votre machine.
Ordicab conserve localement la table de correspondance alias ↔ données réelles. Lors de l'import, il substitue automatiquement chaque alias par la vraie valeur. Le document final est complet et personnalisé — sans que les données réelles aient jamais été transmises à l'IA.

Export IA — étape par étape

Dans la fiche d'un dossier, ouvrez l'onglet Export IA.

  1. Vérifiez les extractions de texte.
    Ordicab indique combien de documents sont extractibles, combien ont été traités, combien restent. Cliquez sur Terminer les extractions manquantes si nécessaire — sans texte extrait, l'IA ne peut pas lire le contenu de ces documents.
  2. Activez la pseudonymisation si nécessaire.
    Cochez Anonymiser les textes et métadonnées exportés pour remplacer les données sensibles par des alias dans le dossier exporté.
  3. Choisissez un répertoire de destination et cliquez sur Exporter pour l'IA. Ordicab crée le dossier structuré à l'emplacement indiqué.
  4. Ouvrez le sous-dossier IA/ avec votre IA Cowork — pas le dossier parent.
    Le dossier exporté contient un répertoire IA/ (données pseudonymisées, contexte, modèles) et un répertoire confidentiel contenant la clé de correspondance et les données PII réelles. Pointez votre agent uniquement sur IA/ pour éviter tout risque de fuite.
    Exemple avec Claude Code : ouvrez un terminal dans IA/ et lancez claude. L'agent lit le fichier de contexte et connaît immédiatement le dossier.
  5. Demandez à l'IA de créer des documents.
    L'agent dépose les fichiers créés dans IA/. Revenez dans Ordicab pour les importer.

Import IA — étape par étape

Dans la fiche du même dossier, ouvrez l'onglet Import IA.

  1. Cliquez sur Choisir le répertoire source et sélectionnez le dossier IA exporté dans lequel l'agent a travaillé.
  2. Ordicab analyse le contenu et liste les fichiers nouveaux ou modifiés.
  3. Sélectionnez les fichiers à importer (ou Tout sélectionner) et cliquez sur Importer les fichiers sélectionnés.

À l'import, Ordicab :

  • restitue les pseudonymes par les données réelles du dossier ;
  • résout les macros Ordicab ({{ }}) en injectant les valeurs correspondantes ;
  • indexe les fichiers importés dans la liste des documents du dossier.

Extraire le texte avant d'exporter

L'extraction de texte est l'opération qui rend le contenu de vos documents lisible par l'IA. Sans elle, l'agent ne voit que les noms de fichiers, pas leur contenu.

Pour extraire : dans l'onglet Documents du dossier, cliquez sur Tout extraire. Ordicab traite PDF, Word, emails et autres formats extractibles. L'opération peut prendre quelques minutes selon le volume.
