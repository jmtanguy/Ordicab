Intelligence artificielle

IA Cowork

Le mode Cowork connecte Ordicab à un agent IA — Claude Code ou OpenAI Codex — qui accède directement aux fichiers de vos dossiers. C'est le mode le plus puissant : l'agent peut lire, analyser et créer des documents en toute autonomie, piloté par vos instructions en langage naturel.

Principe

Un agent Cowork est un programme IA capable de lire et écrire des fichiers sur votre ordinateur. Ordicab lui fournit un contexte structuré sur chaque dossier — contacts, dates clés, références, textes extraits des documents — et l'agent peut ensuite exécuter des tâches complexes :

  • créer un courrier, un contrat ou un rapport à partir d'un modèle Ordicab ;
  • analyser plusieurs pièces d'un dossier et en extraire un résumé ;
  • mettre à jour les métadonnées d'un dossier entier en une seule opération ;
  • comparer des documents, détecter des incohérences, proposer des modifications.

Configurer le mode Cowork

  1. Installez l'agent de votre choix sur votre machine (voir tableau ci-dessus). Un compte actif chez Anthropic ou OpenAI est requis.
  2. Dans Ordicab, ouvrez Paramètres → Configuration IA et sélectionnez Claude Cowork ou OpenAI Codex dans la liste des modes.
  3. Ajoutez le domaine Ordicab à l'application IA. Pour que l'agent connaisse la structure de vos dossiers, vous devez pointer l'agent vers votre répertoire de domaine au premier usage — par exemple en lançant claude dans ce répertoire depuis un terminal, ou en ouvrant ce dossier depuis l'interface de l'agent.
  4. Ordicab confirme la connexion et active les fonctionnalités IA dans l'interface.
Données transmises à l'agent.
Claude Code et Codex envoient vos commandes et le contexte de vos dossiers à leurs serveurs respectifs (Anthropic, OpenAI). Vérifiez que cela est compatible avec vos obligations de confidentialité professionnelle. Pour les données sensibles, privilégiez l'Export IA avec pseudonymisation.

Comment Ordicab prépare le contexte

Ordicab maintient dans chaque dossier un espace caché .ordicab/ contenant des fichiers de configuration lisibles par l'agent :

  • dossier.json — métadonnées du dossier (statut, type, information)
  • contacts.json — liste des contacts et leurs coordonnées structurées
  • key-dates.json — dates clés et leurs notes
  • key-references.json — références clés
  • CLAUDE.md — fichier d'instructions que Claude lit automatiquement au démarrage

L'agent n'a besoin d'aucune configuration supplémentaire — il lit ce contexte et comprend immédiatement la structure du dossier.

L'intent inbox — écriture sans conflit

Lorsque l'agent écrit dans un dossier Ordicab, il ne modifie pas directement les fichiers de données — il dépose ses modifications dans une boîte d'intentions (intent inbox).

Ordicab surveille cette boîte en temps réel. Lorsqu'une intention est détectée, il :

  1. valide que les données respectent les contrats attendus ;
  2. applique les modifications de manière atomique, sans risque de corruption ;
  3. notifie l'interface si une validation échoue — vous pouvez alors corriger manuellement.
Pourquoi ce système est innovant.
Les agents IA travaillent de façon asynchrone et peuvent écrire plusieurs modifications en rafale. L'intent inbox élimine les problèmes de concurrence et de corruption de données qui surviendraient si l'agent écrivait directement dans les fichiers pendant qu'Ordicab les lit. Chaque modification est vérifiée avant d'être appliquée.

Ce que vous pouvez demander à l'agent

L'onglet Assistant IA d'Ordicab contient une référence des opérations disponibles, avec pour chacune un prompt prêt à l'emploi pré-rempli avec le contexte de votre dossier actif. Copiez le prompt, collez-le dans votre session agent, complétez les crochets et envoyez.

Exemples d'opérations :

  • Configurer un dossier entier depuis un texte décrivant le client
  • Extraire les dates clés d'un jugement ou d'un contrat
  • Générer un courrier à partir d'un modèle Ordicab
  • Annoter et taguer les documents d'un dossier
  • Ajouter ou modifier un modèle de document
